# 爬取日本驻马来西亚大使馆签证预约空位

目标网站：https://malaysia.rsvsys.jp/
需求：查出**哪天有空位可以预约、哪天没有**。

爬虫脚本：[[visa_slots.py]]（放在 `Projects/Visa/visa_slots.py`，只用 Python 标准库，无第三方依赖）。

---

## 一、网站是怎么运作的（先看懂再爬）

这是一个日本的预约系统（品牌叫「リザエン / Riza-en」，jQuery + CakePHP）。日历上的空位**不是**写死在 HTML 里的，而是选好类别后用 AJAX 动态加载的。所以爬取分三步：

1. **拿身份票据**
   `GET /reservations/calendar`
   → 服务器返回一个会话 cookie（`USERSESSID`）和一个隐藏字段 `_csrfToken`（防伪造令牌）。之后每次请求都要带上这两样，否则被拒（`CSRF token mismatch`）。

2. **请求某个月的日历**
   `POST /ajax/reservations/calendar`
   表单字段：

   | 字段 | 值 | 含义 |
   |---|---|---|
   | `_csrfToken` | 第 1 步拿到的令牌 | 防伪造 |
   | `search` | `exec` | 固定值，触发查询 |
   | `event` | 例如 `12` | 预约**类别**（见下表）|
   | `plan` | 例如 `9` | **申请内容**（类别下的细分）|
   | `disp_type` | `month` | 月视图（也可 `week`/`day`）|
   | `date` | `2026/07/01` | 想看的月份 |

   → 返回 JSON：`{"html": "<这个月的日历片段>"}`

3. **从返回的 HTML 判断空位**
   每个「营业日」格子里有一张小图标 `<img src=".../icon_XXX.svg">`，图标文件名就代表状态：

   | 图标文件名 | 符号 | 日文原文 | 中文含义 |
   |---|---|---|---|
   | `icon_circle.svg` | ○ | 受付中 / Available | **有空位，可预约** |
   | `icon_triangle.svg` | △ | 残りわずか | **快满了，剩一点** |
   | `icon_disabled.svg` | × | 受付終了 / Not available | **约满 / 不开放 / 假日** |
   | （灰色格子，无图标） | — | — | 过去的日期、周末、或还没开放的月份 |

> [!warning] 最大的坑
> 图标的 `alt` 文字属性在这个网站上是**坏的**——所有格子的 `alt` 都写成 "受付終了／ Not available"，不管实际是不是有空位。
> **千万不要看 `alt` 文字判断状态，要看图标的文件名（`icon_circle` / `icon_triangle` / `icon_disabled`）。** 这是逆向时踩到的最大陷阱。

## 二、类别 / 申请内容对照表（event 和 plan 的 ID）

网页上有两个下拉框，第一个选「类别」，第二个选「申请内容」。它们背后是数字 ID：

**第一个下拉框 —— 类别（`event`）**

| event ID | 日文 | 中文 |
|---|---|---|
| `12` | VISA Application to Japan | **赴日签证申请**（旅游、商务等）|
| `6` | ▼受取専用▼ | 只领取（护照/各种证明/户籍国籍申报/My Number卡/海外选民登记）|
| `9` | ▼申請専用▼ | 只申请（护照/证明/申报/My Number卡/海外选民登记）|

**第二个下拉框 —— 申请内容（`plan`），当 `event=12`（签证）时：**

| plan ID | 日文 | 中文 |
|---|---|---|
| `9` | Visa Application for short term stay (<90days) | **短期停留签证（90天以内）** ← 脚本默认 |
| `20` | COE Holder | 持在留资格认定证明书（COE）者 |
| `22` | VISA submission via Travel Agent (max. 5 persons) | 通过旅行社提交（最多5人）|

> 如果以后这些 ID 变了：先跑 `python visa_slots.py --list` 看类别，再用浏览器打开网站、点开「申请内容」下拉框，看每个选项对应的 `plan` 数字。

## 三、怎么用脚本

在 `Projects/Visa/` 目录下运行：

```bash
# 默认：签证短期停留(event=12/plan=9)，查今后 3 个月里"有空位"的日期
python visa_slots.py

# 显示全部营业日（含约满的 ×）
python visa_slots.py --all

# 查今后 6 个月，输出成 JSON（方便程序处理）
python visa_slots.py --months 6 --json

# 列出所有类别 ID
python visa_slots.py --list

# 换成别的申请内容，例如 COE Holder
python visa_slots.py --event 12 --plan 20
```

**参数说明**

| 参数 | 作用 | 默认 |
|---|---|---|
| `--event` | 类别 ID | `12`（签证）|
| `--plan` | 申请内容 ID | `9`（短期停留）|
| `--months` | 从今天起往后扫几个月 | `3` |
| `--all` | 连约满的日子也显示 | 只显示有空位的 |
| `--json` | 输出 JSON | 输出可读文本 |
| `--list` | 只列出类别 ID 然后退出 | — |

**输出示例**（实际跑出来的样子）：

```
event=12 plan=9  scanned 3 month(s)

  2026-07-16  ○  available
  2026-07-17  ○  available
  2026-07-21  ○  available
  ...
  2026-07-13  ×  closed      # 只有 --all 才会显示这些

11 day(s) with openings out of 14 business day(s) shown.
```

- `available` = 有空位（○）
- `few` = 快满（△）
- `closed` = 约满/不开放（×）

## 四、注意事项

- **数据是实时的**：同一天可能这一刻是 ○、下一刻被人订走就变 ×。逆向时就亲眼看到 7/15 从 ○ 变成 × 了。
- **只能看到已开放的月份**：大使馆通常只开放当月及近期，更远的月份返回的是空日历（全灰、0 个图标），这是正常的，不是脚本坏了。
- **签证有截止时间**：网站写明「签证预约的截止是预约日**前 3 个工作日**下午 3 点」，所以太近的日期即使显示也约不上。
- **不用浏览器**：脚本是纯 HTTP 请求，比开浏览器快很多，也不占资源。（逆向定位阶段用过一次无头浏览器看真实渲染，之后就不需要了。）
